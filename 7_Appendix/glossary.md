# Glossary

## Objective

Provide clear definitions for key terms used across the product documentation to ensure consistent understanding.

---

# Core Concepts

| Term                     | Definition                                                                 |
|--------------------------|-----------------------------------------------------------------------------|
| Incident                 | An emergency situation reported via a call requiring evaluation and response |
| Operator                 | Emergency dispatcher responsible for triaging and dispatch decisions         |
| Recommendation           | System-generated suggestion for incident type, severity, and action          |
| Decision                 | Final action taken by the operator (accept, modify, or reject recommendation)|

---

# AI and System Terms

| Term                     | Definition                                                                 |
|--------------------------|-----------------------------------------------------------------------------|
| Signal Extraction        | Process of identifying key indicators (keywords, context, urgency) from input |
| Recommendation Engine    | System component that generates suggested actions based on signals          |
| Confidence Score         | Numerical estimate of how reliable a recommendation is                      |
| Recall                   | Ability of the system to correctly identify critical incidents              |
| Precision                | Accuracy of recommendations without false positives                         |
| Coverage                 | Percentage of calls where the system provides a recommendation              |

---

# Workflow Terms

| Term                     | Definition                                                                 |
|--------------------------|-----------------------------------------------------------------------------|
| Call Intake              | Initial stage where incoming audio is received and processed                |
| Dispatch                 | Execution of response action based on decision                              |
| Feedback Capture         | Recording operator decisions for system learning                            |
| Override                 | Operator rejection or modification of system recommendation                 |

---

# Trust and Safety Terms

| Term                     | Definition                                                                 |
|--------------------------|-----------------------------------------------------------------------------|
| Human-in-the-Loop        | Design approach where operator retains final decision authority             |
| Explainability           | Ability of the system to justify its recommendations                        |
| Auditability             | Ability to trace decisions and system behavior                              |
| Bias                     | Unequal system performance across different inputs or scenarios             |

---

# Business Terms

| Term                     | Definition                                                                 |
|--------------------------|-----------------------------------------------------------------------------|
| Time to Dispatch         | Time taken from call start to responder dispatch                            |
| AI-Assisted Decision     | Decision where operator uses system recommendation                          |
| Override Rate            | Percentage of recommendations rejected or modified                          |
| Decision Consistency     | Variability in decisions across operators                                   |
